# CodeQuest：与 AI Coding 同频的 Roguelike 卡牌游戏

一款 Electron 桌面应用：左侧是你的 Claude Code CLI（被游戏接管的 PTY 终端 + 日志监听），右侧是一款暗黑式难度、Grim Dawn 式局外成长的回合制卡牌 Roguelike。AI 在 coding=战斗推进；AI 等你介入=游戏自动暂停进入"组卡/整备"阶段，强制你回到 CLI 下达指令后游戏才继续。

---

## 一、核心循环（这是整个游戏的灵魂）

```text
 ┌─────────────────────────────────────────────────────────┐
 │  [AI 输出中]  → 战斗回合自动推进，怪物来袭，掉落滚动      │
 │       │                                                  │
 │       ▼ AI 停顿 > N 秒 / 出现提示符等待输入              │
 │  [整备阶段] → 游戏暂停，可组卡/穿装备/加天赋             │
 │       │                                                  │
 │       ▼ 玩家在 CLI 中回车提交新指令                      │
 │  [战斗继续] → 下一波敌人，节奏与 AI tokens/秒 挂钩        │
 └─────────────────────────────────────────────────────────┘
```

- **AI 忙 = 游戏忙**：tokens/秒 越快 → 出牌速度、爆率、连击数越高
- **AI 闲 = 强制 BREAK**：屏幕中央弹出"⏸ AI 等你回话"，玩家必须切到 CLI 输入才解锁
- 鼓励"投喂高质量 prompt"，因为 AI 一口气产出越多 → 战斗 combo 越长 → 暗金掉率越高

---

## 二、CLI 监控（双通道，自动 fallback）

1. **PTY 包装模式（首选）**：游戏内置一个 xterm.js 终端，用 `node-pty` 启动 `claude` CLI。完整捕获 stdout、ANSI、提示符，最准确判断"输出中 / 等待输入"。
2. **日志监听模式（兜底）**：监听用户指定的会话日志文件 / tmux pipe，按"最后写入时间 + 静默阈值"判断状态。适合已经在外部跑着的 Claude Code。
3. **多窗口聚合**：可同时挂载 N 个 CLI 标签页，任一在输出 → 战斗推进；全部静默 → 进入整备。

状态机输出三个信号：`STREAMING` / `IDLE_WAITING` / `ERROR`，游戏层订阅。

## 三、玩法：回合波次卡牌 Roguelike

- **每波 = AI 一段连续输出**。AI 一停，本波结算。
- **手牌**：攻击 / 法术 / 防御 / 词缀强化牌。每回合抽 5 张，能量 = 上一段 AI 输出 token 数 / 100（封顶）。
- **敌人**：暗黑式三档稀有度（普通/精英/Boss），每 N 波刷一个 Boss。难度曲线对标暗黑——精英带词缀，Boss 多阶段。
- **掉落**：白/蓝/黄/绿/暗金 五档；词缀池含"+X% 暴击""击杀回血""连锁闪电"等约 60 条。
- **库存**：6 格装备槽 + 网格背包，可对比、分解为"以太碎片"（局外货币）。

## 四、成长系统

**局内 Roguelike Build**
- 每波结算三选一卡牌；每 5 波三选一被动；Boss 后必出一件装备。
- 词缀可叠加，鼓励 build 路线（火焰流 / 召唤流 / 暴击流 / 吸血流）。

**局外天赋树（Grim Dawn 风）**
- 每局结束按"AI 实际产出代码行数 + 击杀分数"换算永久天赋点。
- 多个职业盘可同时投点：`战士 / 法师 / 工程师（与 coding 主题呼应）`，盘与盘之间共享中央枢纽节点。
- 提供洗点（消耗以太碎片）。

**装备与库存**
- 暗黑式 tooltip、稀有度颜色、套装效果（2/4/6 件触发）。
- 仓库跨局共享，分解 / 重铸 / 词缀转移三种工艺台。

## 五、与 Coding 行为的挂钩（让"写代码"真的变爽）

- **连续输出 combo**：AI 不间断输出每 30 秒 +1 层 combo，提升掉率与暴击。
- **质量 buff**：在 CLI 中检测到 `git commit` / 测试通过关键字 → 触发"灵感爆发"，下一波必出黄装。
- **介入惩罚**：AI 报错且玩家长时间不回 → 怪物获得"狂暴"层数，逼你回去处理 CLI。
- **专注模式**：可设定"必须 AI 输出 ≥ X 分钟才允许进商店"，强化心流。

## 六、首版范围（MVP）

包含：
1. Electron 外壳 + 内嵌 xterm.js 终端 + node-pty PTY 包装
2. 日志文件监听 fallback（用户在设置里指定路径）
3. CLI 状态机（STREAMING / IDLE / ERROR）+ tokens/秒 估算
4. 卡牌战斗核心：抽牌、能量、出牌、敌人 AI、波次推进、Boss
5. 装备掉落 + 库存 + 5 档稀有度 + 30+ 词缀
6. 局外天赋树（1 个职业盘起步，约 40 节点） + 以太碎片货币
7. 存档（本地 JSON，跨局持久化天赋/仓库/碎片）
8. 设置面板：CLI 路径、静默阈值、键位、音效

不在首版：多人、Steam 工坊、云存档、第二/三职业盘、成就系统。

## 八、技术细节（给开发参考）

- **Stack**：项目已是 TanStack Start + React 19。计划在同仓库新增 `electron/` 目录，用 `@electron/packager` 打包；Vite `base: './'`。
- **PTY**：`node-pty`（Electron main 进程），通过 IPC 把数据流给 renderer 的 xterm.js 与游戏状态机。
- **状态机**：renderer 端 zustand store，订阅 PTY 事件 + 日志 watcher（`fs.watch` + 节流）。
- **游戏渲染**：纯 React + Tailwind（已配置 v4 + shadcn 组件库），战斗动画用 Framer Motion；不引入额外引擎以减小包体。
- **存档**：`app.getPath('userData')/save.json`，原子写入。
- **打包**：Linux `.tar.gz` / macOS `.zip` / Windows `.zip`（沙箱不支持 dmg/exe 安装器）。

## 九、里程碑（按顺序实施）

1. Electron 外壳 + xterm.js 终端 + node-pty 跑通 `claude`，状态机能正确识别 streaming/idle

3. 卡牌战斗 MVP（抽牌/出牌/敌人/波次/Boss），与状态机联动暂停/继续
4. 掉落 + 库存 + 词缀系统
5. 局外天赋树 + 以太碎片 + 存档
6. 平衡、音效、打包脚本、README

完成后你会得到一个可在桌面运行的 `.tar.gz/.zip`，挂着 Claude Code 写代码的同时左手摸鱼右手爆装备。
